OU rode: supabase functions deploy send-free-registration-email --no-verify-jwt

Esta versão SEMPRE responde HTTP 200 → o app NÃO mostra FunctionsHttpError no console.
Se o e-mail falhar (Resend), o corpo vem { success: false, error: "..." }.

--- Verificação 403 resend_rejected (from / domínio) ---
- Invocations 200 = função rodou OK; NÃO significa e-mail enviado.
- 403 validation_error no "from": o e-mail do remetente tem que ser @domínio VERIFICADO na Resend (mesma conta da API key).
- Supabase Secrets:
  - FREE_EVENTS_FROM_EMAIL = só o endereço, SEM aspas no valor. Ex: ingressos@seudominio.com
  - Ou deixe VAZIO para usar onboarding@resend.dev (só teste; destinatário às vezes restrito).
- Deploy de novo após mudar secret não é obrigatório; secrets são lidos a cada invoke.
- Logs da função: linha "from suffix: @seudominio.com" confirma qual domínio está indo na API.
